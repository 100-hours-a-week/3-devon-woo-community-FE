✅ 게시글 상세 페이지 프론트엔드 요구사항 정의서
1) 기능 요약
기능	설명
목록으로 이동	상단 버튼 클릭 → /posts 이동
게시글 수정 이동	수정 버튼 클릭 → /posts/{id}/edit
게시글 삭제	삭제 버튼 클릭 → 삭제 확인 모달 → 삭제 API → /posts 이동
좋아요 기능	좋아요 버튼 클릭 시 좋아요 증가/취소
댓글 목록 조회	댓글 리스트 표시
댓글 작성	텍스트 입력 후 등록
댓글 수정	수정 클릭 시 기존 텍스트 로드 → 수정 모드 전환
댓글 삭제	삭제 버튼 클릭 → 확인 모달 → 삭제 처리
2) UI/UX 규칙
▶ 모달 동작
상태	설명
모달 열림	배경 50% 반투명 블랙, 스크롤/클릭 불가
모달 확인 클릭	삭제 실행
모달 취소/닫기	모달 종료, 스크롤 다시 가능
▶ 좋아요 버튼 상태
상태	색상	클릭 시 동작
disable (비활성, D9D9D9)	#D9D9D9	클릭 → 활성색 #ACA0EB, +1 증가
enable (활성, ACA0EB)	#ACA0EB	클릭 → 비활성색 #D9D9D9, -1 감소

좋아요 숫자 포맷 동일 (1k, 10k, 100k)

▶ 댓글 입력 필드
조건	버튼 색상
입력값 존재	#ACA0EB ➝ #7F6AEE (hover)
입력값 없음	#7F6AEE ➝ #ACA0EB (비활성)
3) 댓글 수정
단계	설명
댓글 수정 클릭	기존 text input에 값 세팅, 버튼 텍스트 댓글 수정으로 변경
수정 완료 버튼 클릭	댓글 API 업데이트 → UI 반영
4) 데이터 포맷/표시 규칙
항목	규칙
숫자 단위	1k / 10k / 100k 표기
댓글 입력	TEXT 저장
조회·댓글·좋아요 표시	게시글 정보와 함께 렌더링
5) API 연동 요구
기능	API
게시글 가져오기	GET /posts/{id}
게시글 삭제	DELETE /posts/{id}
좋아요	POST /posts/{id}/like / DELETE /posts/{id}/like
댓글 조회	GET /posts/{id}/comments
댓글 작성	POST /posts/{id}/comments
댓글 수정	PUT /comments/{commentId}
댓글 삭제	DELETE /comments/{commentId}

요청 중 버튼 중복 클릭 방지 처리

6) 상태 관리 요구
항목	설명
전역 상태	로그인 사용자 정보
로컬 상태	게시글, 댓글 리스트, 좋아요 상태, 모달 상태, 입력 텍스트
에러 처리	alert 또는 메시지 표시
로딩 처리	버튼 로딩 또는 스피너
