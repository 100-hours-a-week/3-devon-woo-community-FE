✅ 회원가입 화면 프론트엔드 요구사항 정의서
1) 화면 요소
요소	설명
프로필 이미지 업로드	업로드/삭제 가능, 업로드 시 미리보기
이메일 입력	helper text 존재 (조건 미충족 시 노출)
비밀번호 입력	유효성 검사, 안내 메시지
비밀번호 확인	일치/불일치 메시지
닉네임 입력	길이 제한 및 유효성 검사
회원가입 버튼	조건 충족 시 활성화
로그인 페이지 이동 링크	/login 이동
2) 입력 검증
✅ 이메일
조건	메시지
빈값	이메일을 입력해주세요
잘못된 형식	올바른 이메일 주소 형식을 입력해주세요 (예: example@example.com)
중복	중복된 이메일 입니다

정규식

/^[^\s@]+@[^\s@]+\.[^\s@]+$/

✅ 비밀번호
조건	메시지
빈값	비밀번호를 입력해주세요
조건 불만족(8~20자, 대문자/소문자/숫자/특수문자 각 1개 이상)	비밀번호는 8자 이상, 20자 이하이며 대문자, 소문자, 숫자, 특수문자를 포함해야 합니다

정규식

/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[!@#$%^&*]).{8,20}$/

✅ 비밀번호 확인
조건	메시지
빈값	비밀번호를 한번더 입력해주세요
불일치	비밀번호가 다릅니다
✅ 닉네임
조건	메시지
빈값	닉네임을 입력해주세요
1자 이하	닉네임은 2글자 이상 입력해주세요
10자 초과	닉네임은 최대 10자까지 작성 가능합니다
중복	중복된 닉네임 입니다
3) 버튼 조건
상태	조건	버튼 색상
비활성	하나라도 미입력/검증 실패	#ACA0EB
활성	모든 입력 valid	#7F6AEE

활성 버튼 클릭 → 회원정보 저장 → 로그인 페이지 이동

4) 프로필 이미지 업로드
기능	설명
사진 업로드 클릭	파일 탐색기 열림
업로드 성공 시	미리보기 출력
이미지 재업로드 시	이전 이미지 대체
삭제 동작	삭제 후 기본 이미지 표시
5) 사용자 흐름
동작	결과
회원가입 버튼 클릭 (성공)	회원정보 저장 → /login 이동
로그인 버튼 클릭	/login 이동
동그라미 프로필 클릭	이미지 업로드
