📋 기능 요구사항
1️⃣ 비밀번호 입력 및 유효성 검사

설명: 비밀번호 입력란에 올바른 비밀번호가 입력되지 않거나, 유효성 검사를 통과하지 못할 경우 오류 메시지를 출력한다.

유효성 조건:

비밀번호는 8자 이상, 20자 이하

대문자, 소문자, 숫자, 특수문자를 각각 최소 1개 이상 포함해야 함

세부 동작:

비밀번호 입력란(password input)

미입력 시: *비밀번호를 입력해주세요.

확인 입력란과 불일치 시: *비밀번호 확인과 다릅니다.

비밀번호 확인 입력란(password confirm input)

미입력 시: *비밀번호를 한 번 더 입력해주세요.

비밀번호 불일치 시: *비밀번호와 다릅니다.

비고:

실시간 유효성 검사(입력 시 즉시 피드백) 구현 권장.

백엔드에서도 동일 규칙으로 유효성 검증 수행 필요.

2️⃣ 버튼 활성화 조건 및 수정 처리

설명: 모든 입력 필드가 채워지고 유효성 검사를 통과했을 때만 수정하기 버튼이 활성화된다.

동작 조건:

기본 버튼 색상: #ACA0EB

활성화 시 색상 변경: #7F6AEE

수정 프로세스:

유효성 검사 통과 후 “수정하기” 버튼 클릭

비밀번호 변경 요청 API 호출

성공 시 Toast 메시지 출력 → “수정 완료”

필요 시 로그인 페이지로 리다이렉트 (선택)
