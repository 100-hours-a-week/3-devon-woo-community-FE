📋 기능 요구사항
1️⃣ 제목 입력 제한

설명: 게시글 제목은 최대 26자까지만 입력 가능해야 한다.

제한 조건:

27자 이상 입력 시 작성 불가능.

프론트엔드에서 입력 제한 처리 필요.

비고:

실시간 글자 수 카운팅 기능 고려 가능(선택 사항).

2️⃣ 게시글 본문 저장 방식

설명: 게시글 본문은 데이터베이스에 LONGTEXT 타입으로 저장되어야 한다.

DB 컬럼 타입: LONGTEXT

비고:

대용량 텍스트 저장용으로 설정.

HTML 태그 포함 여부는 정책에 따라 결정.

3️⃣ 이미지 파일 처리

설명: 수정 페이지 진입 시 기존 업로드된 이미지 파일이 표시되어야 한다.

제한 조건:

이미지 파일은 1개만 업로드 가능.

기존 이미지를 교체할 경우 새 파일 업로드 시 이전 파일은 자동으로 삭제 혹은 교체 처리.

비고:

허용 확장자: jpg, png, gif 등

파일 용량 제한 정책 필요.

4️⃣ 수정 및 페이지 이동

설명: 수정 버튼 클릭 시 게시글 수정이 완료되고, 해당 게시글의 상세보기 페이지로 이동해야 한다.

동작 시나리오:

사용자가 제목, 본문, 이미지를 수정 후 “수정” 버튼 클릭

유효성 검증(제목/본문 필수 확인)

수정된 데이터 DB 반영

해당 게시글 상세보기 페이지로 리다이렉션

비고:

수정 완료 시 알림 메시지(예: “게시글이 수정되었습니다.”) 출력 권장.
