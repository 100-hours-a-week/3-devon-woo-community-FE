📋 기능 요구사항
1️⃣ 제목 입력 제한

설명: 게시글 제목은 최대 26자까지만 입력 가능해야 한다.

제한 조건:

27자 이상 입력 시 작성이 불가능해야 함.

클라이언트 단(프론트엔드)에서 입력 제한 처리 필요.

비고:

입력 시 실시간으로 남은 글자 수 표시 기능 고려 가능(선택 사항).

2️⃣ 게시글 본문 저장 방식

설명: 게시글 본문은 데이터베이스에 LONGTEXT 타입으로 저장되어야 한다.

DB 컬럼 타입: LONGTEXT

비고:

대용량 텍스트 저장을 고려한 타입 선택.

HTML 포맷 저장 여부는 별도 정책에 따름.

3️⃣ 이미지 업로드

설명: 클릭 시 컴퓨터에서 이미지 파일을 업로드할 수 있어야 한다.

동작 조건:

이미지 파일 형식만 허용 (jpg, png, gif 등)

업로드 시 미리보기 가능하도록 구현 권장.

비고:

서버 업로드 시 파일 크기 및 확장자 검증 필요.

4️⃣ 버튼 활성화 및 유효성 처리
(1) 버튼 색상 변경 규칙

조건: 제목과 본문이 모두 입력된 경우

동작: 등록 버튼 색상이 **#ACA0EB → #7F6AEE**로 변경됨.

효과: 입력 완료 상태 시 버튼 활성화 시각적 피드백 제공.

(2) 버튼 클릭 시 검증

조건: 제목 또는 본문이 비어 있을 때 버튼 클릭 시

동작:

게시글 등록 불가

경고 메시지 출력:

“제목, 내용을 모두 작성해주세요.”

비고:

입력 검증은 프론트엔드 단에서 수행.

서버 단에서도 동일한 유효성 검증 수행 필요.
