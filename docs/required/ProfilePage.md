📋 기능 요구사항
1️⃣ 프로필 이미지 인터랙션

설명: 프로필 이미지를 클릭 또는 드롭다운하여 변경할 수 있다.

상세 조건:

프로필 이미지 클릭 시 파일 선택 창 또는 드롭다운 메뉴 표시

hover 시 배경색 변경 (#E9E9E9)

클릭 시 각 관련 페이지(예: 마이페이지, 이미지 변경 페이지 등)로 이동 가능

비고:

이미지 교체 즉시 미리보기 반영 필요

허용 확장자: JPG, PNG, GIF

2️⃣ 닉네임 유효성 검증 및 수정 버튼 동작

설명: 수정하기 버튼 클릭 시 닉네임 입력 여부와 중복 여부를 검증한다.

유효성 조건:

닉네임 미입력 시 → *닉네임을 입력해주세요.

닉네임 중복 시 → *중복된 닉네임 입니다.

닉네임 길이 10자 초과 시 → *닉네임은 최대 10자까지 작성 가능합니다.

비고:

모든 검증 실패 시 버튼 동작 중단.

검증 성공 시 수정 프로세스 진행.

3️⃣ 수정 완료 처리

설명: 수정하기 버튼 클릭 시 회원정보 수정이 수행된다.

동작 시나리오:

입력된 닉네임 검증 성공 시 DB에 수정 요청

수정 성공 시 Toast 메시지 출력

“수정 완료”

화면 내 즉시 반영 (프로필, 닉네임 등 업데이트)

비고:

실패 시 에러 메시지 표준화 필요 (예: 서버 오류, 중복 닉네임 등)

4️⃣ 회원 탈퇴 기능

설명: 회원 탈퇴 버튼 클릭 시 확인 모달이 표시된다.

동작 시나리오:

“회원 탈퇴” 클릭 시 확인 모달 표시

확인 클릭 시 탈퇴 완료 처리

탈퇴 완료 후 로그인 페이지로 이동

비고:

탈퇴 처리 전 확인 메시지 표시:

“정말로 탈퇴하시겠습니까? 모든 데이터가 삭제됩니다.”

취소 클릭 시 모달 닫힘.
